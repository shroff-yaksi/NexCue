## Executive Summary

NexCue is a **universal "What's Next?" decision engine** that solves the modern problem of content hoarding and decision fatigue. Instead of scattered saved items across YouTube, Netflix, Instagram, podcasts, bookmarks, wishlists, and more - NexCue centralizes everything and answers: **"What should I do next?"**

## Key Risk: Platform Integration (CRITICAL)

**Reality: User permission ≠ data access. APIs decide, not users.**

### Platform API Availability

| Platform | Saved Items via API? | Safe for SaaS? |
|----------|---------------------|----------------|
| YouTube | ✅ Yes | ✅ Yes |
| Notion | ✅ Yes | ✅ Yes |
| Pocket | ✅ Yes | ✅ Yes |
| Instagram | ❌ No | ❌ No |
| TikTok | ❌ No | ❌ No |
| X/Twitter Bookmarks | ❌ No | ❌ No |
| Netflix/Prime/Hotstar | ❌ No | ❌ No |
| Google Keep | ❌ No | ❌ No |
| Apple Notes | ❌ No (local) | ❌ No |

### The Solution: Share-Based Import

**Correct Approach:** "You choose to send something to my app"

1. User sees content on Instagram/Netflix/etc.
2. Taps Share → Selects "NexCue"
3. NexCue receives the URL
4. Extracts metadata (Open Graph, TMDB)
5. Saves to queue

**This covers:**
- ✅ Instagram reels/posts
- ✅ TikTok videos
- ✅ Netflix shows (via share + TMDB)
- ✅ Any platform with Share button

**Mitigation Strategy:**
1. MVP: Manual add + URL paste (no dependencies)
2. Phase 2: Share extension (iOS/Android)
3. Phase 3: Official APIs only (YouTube, Notion, Pocket)
4. NEVER: Scraping or password-based access

# Recommendations

## Top 10 Recommendations

1. **START WITH MANUAL ADD**
   - Don't wait for integrations
   - URL paste + manual is v1
   - Prove value before complexity

2. **NAIL THE DECIDE FLOW**
   - This is THE differentiator
   - Make it feel magical
   - Iterate based on usage data

3. **BUILD COMPLETION DOPAMINE**
   - Satisfying animations
   - Streak gamification
   - Visible progress

4. **PRODUCT HUNT LAUNCH IS CRITICAL**
   - Prepare 6 weeks ahead
   - Line up 50+ supporters
   - Have launch day plan

5. **CONTENT MARKETING FOR SEO**
   - "Netflix overwhelm" content
   - "Watch Later problem" articles
   - Own the decision fatigue niche

6. **FREEMIUM WITH CLEAR VALUE**
   - Free tier is useful (25 items)
   - Pro value obvious (unlimited + insights)
   - No artificial crippling

7. **CROSS-PLATFORM FROM DAY 1**
   - PWA works everywhere
   - Don't pick iOS vs Android
   - Web-first, mobile-capable

8. **MINIMIZE FRICTION**
   - One-tap decide
   - Zero-config start
   - Skip is always okay

9. **BUILD IN PUBLIC**
   - Twitter/X presence
   - Share weekly updates
   - Engage with indie community

10. **MEASURE COMPLETION, NOT ADDS**
    - North star = items completed
    - Not queue size
    - Not daily opens

## Identified Gaps & Considerations

### Potential Gaps in Current Plan

| Gap | Consideration | Mitigation |
|-----|---------------|------------|
| **Cold start problem** | First-time users have NO items | Sample queue option, quick multi-URL import |
| **Re-engagement** | User stops for 2 weeks → hard to return | Weekly digest email, win-back notifications |
| **Mobile-first reality** | 80% of decisions on phone | PWA initially, native if needed |
| **Habit formation** | New habits are hard to build | Focus on completion dopamine, not guilt |
| **Another app to open** | Adding to app fatigue | Must feel LIGHTER than the problem |
