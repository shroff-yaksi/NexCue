# Platform Integration Reality (CRITICAL)

## The Hard Truth About APIs

**User permission ≠ data access. APIs decide, not users.**

This is why many "universal save" apps fail. Understanding this constraint is essential.

### Platform API Availability

| Platform | Saved Items via API? | User Permission? | Safe for Public App? |
|----------|---------------------|------------------|---------------------|
| **YouTube** | ✅ Yes | ✅ Yes | ✅ Yes |
| **Notion** | ✅ Yes | ✅ Yes | ✅ Yes |
| **Pocket** | ✅ Yes | ✅ Yes | ✅ Yes |
| **Instagram** | ❌ No | ❌ No | ❌ No |
| **TikTok** | ❌ No | ❌ No | ❌ No |
| **X (Twitter)** | ❌ No | ❌ No | ❌ No |
| **Netflix/Prime/Hotstar** | ❌ No | ❌ No | ❌ No |
| **Google Keep** | ❌ No | ❌ No | ❌ No |
| **Apple Notes** | ❌ No (local only) | ❌ No | ❌ No |

### What This Means for the Product

```
+====================================================================================+
|                    THE CORRECT PRODUCT STRATEGY                                    |
+====================================================================================+
|                                                                                    |
|  ❌ WRONG APPROACH (How many apps fail):                                            |
|  "My app reads your saved content from Instagram/Netflix/etc."                      |
|  - Requires scraping = ToS violation                                                |
|  - Temporary at best, banned at worst                                               |
|  - Not scalable for a real business                                                 |
|                                                                                    |
|  ✅ CORRECT APPROACH (NexCue strategy):                                             |
|  "You choose to send something to my app"                                           |
|  - User shares from Instagram → NexCue receives                                     |
|  - No scraping, no ToS violations                                                   |
|  - Fully legal and scalable                                                         |
|  - Works with ANY platform that has "Share" button                                  |
|                                                                                    |
+====================================================================================+
```

### The Share-Based Import Solution

**How it works:**
1. User sees a reel/post on Instagram
2. Taps Share → Selects "NexCue"
3. NexCue receives the URL
4. Extracts metadata (Open Graph, TMDB for movies)
5. Saves as item in queue

**What this covers:**
- ✅ Instagram reels/posts
- ✅ TikTok videos
- ✅ Twitter/X threads
- ✅ Netflix shows (via share link + TMDB lookup)
- ✅ YouTube videos
- ✅ Any shareable content

**Limitation (honest):**
- Cannot import OLD saved collections
- Only works for NEW items going forward
- This is acceptable and common

### Why This Is Actually Better

This share-based model aligns with NexCue's philosophy:

| Benefit | Explanation |
|---------|-------------|
| **Intentional items only** | Only content you consciously send enters the queue |
| **No clutter from old saves** | Fresh start, not importing years of hoarded content |
| **Learning Queue stays light** | Manageable queue from day one |
| **Reduces decision fatigue further** | Pre-filtered by intent |

> **"The place I intentionally send things I want to consume"** is a strength, not a limitation.
